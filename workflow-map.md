# Workflow Map

This file maps a simplified NHS outpatient referral-to-appointment pathway.

## Current pathway

Referral received
→ admin validation
→ clinical triage
→ appointment booked
→ patient notified
→ clinic takes place
→ outcome documented
→ follow-up, discharge or further tests arranged
→ manager reviews waiting list and capacity

## User groups

### Admin / booking team

Main tasks:

* check referral details
* chase missing information
* book appointments
* contact patients
* manage cancellations and DNAs
* update systems or spreadsheets

Possible frustrations:

* unclear referral information
* duplicate data entry
* manual chasing
* pressure from long waiting lists
* lack of visibility on clinic capacity

### Clinicians

Main tasks:

* review referrals
* prioritise patients
* decide correct clinic type
* document clinic outcomes
* request follow-up, tests or discharge

Possible frustrations:

* incomplete referral information
* unclear triage lists
* patients booked into the wrong clinic
* documentation delays
* extra admin around follow-up plans

### Service manager / operational lead

Main tasks:

* monitor waiting list pressure
* understand clinic utilisation
* track delays and bottlenecks
* manage capacity
* report performance

Possible frustrations:

* relying on spreadsheets
* delayed or incomplete data
* limited visibility of where delays happen
* difficulty proving whether changes are improving the pathway

## Main workflow problems

1. Referrals may arrive with missing or unclear information.
2. Admin teams may spend time chasing details manually.
3. Clinical triage can be delayed if worklists are unclear.
4. Patients may be booked into the wrong clinic type.
5. DNAs and cancellations may not be acted on quickly.
6. Clinic outcome documentation may be delayed.
7. Managers may lack a clear view of waiting list pressure and capacity.
8. Staff may keep using old workarounds if a new system feels slower.

## Implementation point

Before any software rollout, I would want to understand how the pathway actually works in practice, not just how it is supposed to work on paper. The important questions are:

* where does work enter the system?
* who touches it first?
* where are delays created?
* what information is missing?
* which users depend on each other?
* what manual workarounds already exist?
* what would make staff trust or reject the new tool?
